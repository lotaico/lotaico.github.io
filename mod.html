Warn System

Lotai has a warn system which you can use to issue anything from simple warnings, to outright banning people.  You can always remind yourself on how to use the warn command by simply running -warn, preferably in private channels such as #staff-chat.

You can issue different warnings based on the severity of the users actions.  The user that is warned won’t be told who warned them, but if you run the command in a non-private channel such as #general, they may see.

All warnings are logged in #mod-log and you can see warnings for a user using -warnings <user> or a full list of warnings on Lotai using -warnlist

You can edit and manage warnings per user using the -warnings <user> command and using the reactions to scroll through the warnings and edit or delete them.

You may notice there are commands such as -kick -ban & -mute, these do not fully integrate with the Warn System and therefore should not be used under any circumstances.  Always use -warn.

You should always warn with a reason, this is for logging purposes and also to inform the user of what they did wrong.

Substitutions for quickly issuing warnings

The Warn System allows for the use of substitutions to quickly write out warnings.  When warning a user, you can use many of the substitutions that you can find using -subs[a].  When using substitutions, you must put them in [square brackets] so the bot knows what is meant to be a substitution.  If you're warning a new user, you can encourage them to have a reread of the rules using the substitution “welc” with [welc].  Here is an example of how it would work.

-warn 2 201620873952624641 5m [s-img] [welc] Last warning.

would be the same as typing

-warn 2 201620873952624641 5m Spam; Posting many images consecutively and/or within a short period. Please read #welcome to become familiar with the rules. Last warning.

Setting durations for punishments

You can set durations for mutes and bans for how long they should last.  In Examples of using the System it explains where you can type in the durations for those types of punishments.  Durations can only be one word so the bot can identify durations in commands.  There are multiple ways of writing and structuring durations that the bot can understand.

You can set a duration to the mute with the first word of the reason, which should be a number followed by the unit. Examples:

20s = 20secs = 20seconds: 20 seconds

5m = 5minutes = 5min: 5 minutes

2h = 2hours = 2hrs: 2 hours

1d = 1day: a day

7d = 7days: a week

They can also be stacked like this to allow for more specific durations:

5m30s: 5 minutes and 30 seconds

1d12h: One day and a half

1h45m: 1 hours and 45 minutes

Examples of using the system

Issuing a warning

Issuing a warning will send a DM to the user notifying them of being warned, along with the reason for being warned.  Warnings are classified as warns with a severity of 1 and can be issued using the following command(s).

-warn <user> <reason>

or

-warn 1 <user> <reason>

Muting a user

Muting someone will send them a DM similar to a warning, notifying them of being muted, along with the duration of the mute (if a duration is set).  Mutes are classified as warns with a severity of 2 and can be issued with the following command(s).

-warn 2 <user> <duration> <reason>
or

-warn 2 <user> <reason>

or

-warn 2 <user>

Kicking a user

Kicking someone will send them a DM similar to a warning, notifying them of being kicked, along with the reason for being kicked.  Kicks are classified as warns with a severity of 3 and can be issued with the following command(s).

-warn 3 <user> <reason>

or

-warn 3 <user>

Soft-banning a user

Soft-bans ban the user, and unban the user instantly afterwards.  It basically kicks the user and deletes the last day of messages from them.  Soft-banning someone will send them a DM similar to a warning, notifying them of being soft-banned, along with the reason for being soft-banned.  Soft-bans are classified as warns with a severity of 4 and can be issued with the following command(s).

-warn 4 <user> <reason>

or

-warn 4 <user>

Banning a user

Banning someone will send them a DM similar to a warning, notifying them of being banned, along with the duration of the ban (if a duration is set).  When the ban duration is up, or the user is unbanned, the bot will attempt to send a DM to the user containing an invite back to Lotai.  Bans are classified as warns with a severity of 5 and can be issued with the following command(s).

-warn 5 <user> <duration> <reason>

or

-warn 5 <user> <reason>

or

-warn 5 <user>

Cleanup

Cleanup is an advanced tool that you can use for clearing messages.  You can cleanup messages based on parameters that you provide.  You can always remind yourself on how to use the -cleanup command by running -cleanup

What to clean

Delete the last <x> messages.

-cleanup messages <x>

Delete the last <x> messages from a specified user.

-cleanup user <user> <x>

Delete the last <x> messages matching the specified text.

-cleanup text “text”[b] <x>

Delete all messages after a specified message.

-cleanup after <message ID>

Delete the messages between Message One and Message Two.

-cleanup between <ID 1> <ID 2>

Deletes <x> messages before the specified message. ⚠️Destructive command

-cleanup before <message ID>

Other Tools

Server and Channel locking

You can lock channels, including voice channels using the commands listed below.  The bot does not know the state the channel is in and instead overwrites channel permissions depending on what action you command (unlock or lock).  You can lock text channels separately, but voice channels must all be locked at the same time.  Channels that are locked only allow moderators to type / speak.  Locking should be used in case of a raid or in case of excessive spam.

Locking

Unlocking

What it does

-lock

-unlock

Will lock / unlock the channel the command is run in.

-lockvc

-unlockvc

Will lock / unlock all voice channels, not including Event (Event channel will not have any permissions changed.)

-lockserver

-unlockserver

Will lock all text channels and voice channels.  This command takes a second or two as it changes permissions for every channel not including tickets and staff channels.

Slowmode

Set the slowmode of a channel with -slowmode <seconds>.  You can deactivate slowmode by running -slowmode.  This will set a slowmode using discords native slowmode feature.  You can set a slowmode in case of spam (especially with things such as sai ke tau ilo)

Lockdown

Users can be put into “lockdown” in which they cannot access many parts of the server and have access to a channel that states that their account is currently locked down.  You can use this for new accounts that you are suspicious about, or users you think may be harmful toward the community.  You have full rights to put anyone into lockdown without reason, and then contact me if you need any help.  Using the command is as follows

-lockdownuser <user>

Tickets[c]

This section only applies to Ticket Managers, so if you are just a moderator that doesn’t manage tickets, you can ignore this section.

Tickets are used for all inquiries, suggestions and concerns regarding Lotai and / or its moderators.  Tickets create a private space for the ticket creator and ticket managers which include me, and those capable of handling tickets.   Tickets allow for things to be more organized, manageable and easy to look at later.  As a ticket manager, you can close tickets, for when no extra input is required or if the open requests the ticket be closed.  Tickets can always be opened later for any reason.  To close tickets, run -ticket close in the ticket you want to close.

If you feel as if your not qualified to handle a certain ticket, (example: a ticket with a bot suggestion) you can assign it to me with -assignticket @sexnine#6969

Additionally, I may assign a ticket to you if I cannot handle it at that moment or if it is concerning you specifically.  If you ever need any help with a ticket, you can always ping me in #staff-chat

To categorize tickets or update their status, you can use -ticket update <status>
